# 📘 MWAgent Service Implementation Guide (Expert Version)

> **문서 개요**:  
> 본 문서는 `MWAgent` 프로젝트의 **서비스화(Service Wrapper)** 및 **Standalone 실행 환경 구축** 과정을 상세히 기술한 기술 문서입니다.  
> 주어진 리소스(JAR 파일 등)를 바탕으로 초기 분석부터 환경 구축, 트러블슈팅, 최종 배포 스크립트 작성까지의 전 과정을 '후임자가 처음부터 다시 구축할 수 있는 수준'으로 자세히 기록합니다.

---

## 1. 프로젝트 배경 및 초기 분석 (Initial Analysis)

### 1.1. 주어진 리소스 (Input)
프로젝트 시작 시점에 우리가 받은 파일들은 다음과 같았습니다.
*   **`mwmanger-0.9.10.jar`**: 핵심 로직이 담긴 Java 애플리케이션 (Main Jar)
*   **`lib/`**: 의존성 라이브러리 디렉토리
*   **목표**: 이 Java 프로그램을 **Windows Service**로 등록하여, 서버 부팅 시 자동 실행되게 만드는 것.

### 1.2. 기술적 제약 사항 (Constraints)
*   **Java Application**: `.exe` 실행 파일이 아니라 `.jar` 파일이므로, Windows Service Manager(SCM)에 직접 등록할 수 없음.
*   **Service Wrapper 필요**: 따라서 `WinSW` (Windows Service Wrapper)와 같은 중재자 프로그램이 필요함.
*   **배포 환경의 변화**: 초기엔 서비스 등록을 목표로 했으나, 추후 배포 환경 제약으로 인해 **Pure Java Script (BAT)** 방식도 지원해야 했음.

## 2. 개발 과정 (Construction Phase)

### 2.1. WinSW(Windows Service Wrapper) 구성
Java 프로그램을 서비스로 돌리기 위해 `mwagent-service.exe` (WinSW 리네임)와 `mwagent-service.xml` 설정 파일을 구성했습니다.

#### 핵심 설정 파일: `mwagent-service.xml`
이 파일이 서비스의 '심장'입니다. JVM을 어떻게 구동할지 정의합니다.

```xml
<service>
  <id>MWAgent</id>
  <name>MWAgent Service</name>
  <executable>java</executable>
  <arguments>-cp .;lib/*;mwmanger-0.9.10.jar -Dname=mwagent.SYSADMIN -Xms256m -Xmx256m mwmanger.MwAgent</arguments>
  
  <env name="USER" value="SYSADMIN"/>
  <env name="HOSTNAME" value="WIN-RVA6N8ACU27"/>
  <env name="DOMAIN_NAME" value="SCP_WINDOW"/>
</service>
```

### 2.2. 시행착오 및 트러블슈팅 (Troubleshooting) 🛠️

#### 🔴 이슈 1: Double Slash URL 문제 (HTTP Connection Error)
*   **증상**: 서비스가 시작되자마자 종료됨.
*   **원인**: `agent.properties`의 `server_url` 끝에 슬래시(`/`)가 있어서 코드 내부에서 URL 합칠 때 이중 슬래시 발생.
*   **해결**: `server_url`의 마지막 슬래시 제거.

#### 🔴 이슈 2: 인증 토큰 만료 (422 Unprocessable Entity)
*   **증상**: `Response code: 422` 에러 발생.
*   **원인**: 서버 측에서 토큰 거부.
*   **해결**: 새로운 유효 토큰을 발급받아 `agent.properties`의 `token` 값 갱신.

#### 🔴 이슈 3: 환경 변수 누락과 ProcessBuilder
*   **증상**: 수동 실행 시 `host_id does not exist` 에러 발생.
*   **원인**: 수동 실행 시 `HOSTNAME` 환경변수 누락.
*   **해결**: 수동 실행 스크립트(bat)에도 `set HOSTNAME=...` 구문 추가.

## 3. 배포 환경 전환 (Service → Standalone)

프로젝트 후반부, "Windows Service를 사용할 수 없는 배포 환경"이라는 추가 요구사항이 발생하여 **Pure Java 실행 스크립트**를 개발했습니다.

### 3.1. Start Script (`start_mwagent.bat`)
`mwagent-service.xml`의 설정을 그대로 옮겨온 배치 파일입니다.

### 3.2. Stop Script (`stop_mwagent.bat`)
WMIC를 사용하여 Command Line에 특정 문자열(`-Dname=mwagent.SYSADMIN`)이 포함된 프로세스만 골라 종료합니다.
*   **왜 `taskkill /IM java.exe`를 안 썼나?**: 다른 Java 프로그램까지 죽일 수 있기 때문.

## 4. 유지보수 가이드
1.  **서버 주소 변경 시**: `agent.properties`의 `server_url` 수정 (끝에 슬래시 금지)
2.  **토큰 갱신 시**: `agent.properties`의 `token` 값 교체
3.  **메모리 부족 시**: `-Xmx256m` 수치를 상향 조정
4.  **로그 확인**: `mwagent.0.0.log`의 마지막 50줄부터 확인
