# MWAgent 배포 및 운영 가이드

본 가이드는 개발 완료된 MWAgent를 실제 테스트 서버에 배포하고 운영하기 위한 절차를 안내합니다.

## 1. 배포 파일 준비 (Local)

### ✅ 필수 포함 항목
| 구분 | 항목 | 설명 |
| --- | --- | --- |
| **실행 파일** | `mwagent.jar` | 에이전트 핵심 파일 |
| **라이브러리** | `lib/` | 의존성 라이브러리 폴더 전체 |
| **설정 파일** | `agent.properties` | 서버 연동 설정 (배포 후 수정 필요) |
| **스크립트** | `start_mwagent.bat` | 수동 실행 스크립트 |
| | `stop_mwagent.bat` | 수동 중지 스크립트 |
| | `run.agent.bat` | 자동 업데이트/실행 스크립트 |
| | `restart.bat` | 재시작 스크립트 |
| **서비스 (옵션)** | `mwagent-service.xml` | 윈도우 서비스 등록 설정 |
| | `mwagent-service.exe` | 윈도우 서비스 래퍼 실행 파일 |

> **Note**: `log` 폴더와 `_old` 폴더는 복사하지 않아도 됩니다.

---

## 2. 서버로 파일 전송 (Deploy)

- **권장 경로**: `C:\MWAgent` 또는 `C:\Users\{UserName}\MWAgent`
- **주의**: 경로에 공백이나 한글이 포함되지 않는 것이 좋습니다.

---

## 3. 서버 설정 변경 (Configuration)

### 3.1 `agent.properties` 수정
- `server_url`: 실제 운영 서버의 주소로 변경 필요
- `token`: (필요 시) 서버에서 발급받은 새로운 토큰으로 교체

### 3.2 `mwagent-service.xml` 수정 (서비스 사용 시)
- `<env name="USER" value="SYSADMIN"/>`: 서버의 실제 사용자명으로 변경
- `<env name="HOSTNAME" value="..."/>`: 서버의 호스트명으로 변경
- `<workingdirectory>`: 실제 설치된 경로로 변경

---

## 4. 실행 및 점검 (Execution)

### 방법 A: 수동 실행 (디버깅 용도)
1. `cmd` 창을 엽니다.
2. 설치 경로로 이동합니다.
3. `start_mwagent.bat`를 실행합니다.

### 방법 B: 윈도우 서비스 등록 및 실행 (운영 용도)
1. 관리자 권한으로 `cmd`를 엽니다.
2. `mwagent-service.exe -i mwagent-service.xml` 명령어로 서비스 등록.
3. `net start MWAgent` 또는 서비스 관리자에서 시작.

---

## 5. 운영 및 업데이트

### 업데이트 방법 (새 버전 배포 시)
```cmd
run.agent.bat mwagent-new-version.jar
```
명령어를 실행하면 **자동으로 백업, 교체, 재시작**됩니다.
