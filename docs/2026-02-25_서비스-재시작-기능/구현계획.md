# MWAgent 서비스 자체 재시작 기능 구현 (방법 3: Windows 작업 스케줄러)

MWAgent가 MW Server에서 업데이트 지시를 받으면, `run.agent.bat`을 호출하여 **Windows 작업 스케줄러**에 재시작 작업을 등록하고, 실제 재시작은 `restart.bat`이 수행하는 구조입니다.

## 동작 흐름

1. **MWAgent (Java)** → `run.agent.bat` 호출 (선택적: 새 JAR 경로)
2. **run.agent.bat** → 새 JAR 스테이징 + `MWAgentRestart` 작업 등록 + 즉시 실행
3. **restart.bat** (작업 스케줄러에 의해 별도 프로세스) → 프로세스 중지 → JAR 교체 → 재시작 → 스케줄러 작업 삭제

## 변경된 파일

### run.agent.bat
- 기존 JAR 업데이트 스크립트 → **작업 스케줄러 등록 스크립트**로 전면 교체
- 인자 없음: 단순 재시작 / 인자 있음: JAR 교체 후 재시작
- `new_mwagent.jar`로 스테이징 → `schtasks /Create` + `/Run`

### restart.bat
- 기존 단순 재시작 → **JAR 업데이트 + 스케줄러 정리** 추가
- `new_mwagent.jar` 존재 시: 기존 JAR 백업 → 교체
- 완료 후 `MWAgentRestart` 스케줄러 작업 삭제

## 테스트 방법

| 시나리오 | 명령어 | 확인사항 |
|---------|--------|---------|
| 단순 재시작 | `run.agent.bat` | 스케줄러 등록 → 재시작 → 작업 삭제 |
| JAR 업데이트 | `run.agent.bat new.jar` | 스테이징 → 백업 → 교체 → 재시작 |
| 에러 처리 | `run.agent.bat xxx.jar` | 파일 없음 에러, 스케줄러 미등록 |
