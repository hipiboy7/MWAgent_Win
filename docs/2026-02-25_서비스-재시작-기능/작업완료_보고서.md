# MWAgent 서비스 기반 스크립트 통일 완료

## 변경 요약

| 파일 | 변경 내용 |
|------|-----------|
| `restart.bat` | `stop/start_mwagent.bat` 호출 → `net stop/start MWAgent` (서비스 제어)로 변경 |
| `start_mwagent.bat` | 디버깅 전용 주석 추가 (코드 변경 없음) |
| `stop_mwagent.bat` | 디버깅 전용 주석 추가 (코드 변경 없음) |
| `run.agent.bat` | 변경 없음 |

## 최종 파일 역할 정리

| 파일 | 역할 | 사용 환경 |
|------|------|-----------|
| `mwagent-service.exe` + `.xml` | Windows 서비스 래퍼 + 설정 | 운영 |
| `mwagent.jar` | Java 에이전트 프로그램 | 운영 |
| `agent.properties` | 서버 URL, 토큰 등 앱 설정 | 운영 |
| `run.agent.bat` | 재시작 트리거 (스케줄러 등록) | 운영 (MWAgent Java가 호출) |
| `restart.bat` | 서비스 재시작 실행 (`net stop/start`) | 운영 (스케줄러가 호출) |
| `start_mwagent.bat` | 콘솔 모드 직접 실행 | 디버깅 전용 |
| `stop_mwagent.bat` | 프로세스 직접 종료 | 디버깅 전용 |

## 전체 재시작 흐름

```
MWAgent Java → run.agent.bat [jar] → 작업 스케줄러 등록
                                          ↓
                                     restart.bat 실행
                                          ↓
                               net stop MWAgent (서비스 중지)
                                          ↓
                              JAR 교체 (new_mwagent.jar 있으면)
                                          ↓
                               net start MWAgent (서비스 시작)
                                          ↓
                                스케줄러 작업 삭제 (정리)
```

## 참고사항

### JAR 다운로드 방식 (확정)
- MWAgent는 서버에서 새 JAR을 **`mwagent.jar`로 직접 다운로드** (덮어쓰기) → `run.agent.bat` 인자 없이 호출 → 서비스 재시작
- 리눅스/유닉스 환경과 동일한 방식

### 파일 잠금(File Lock) 관련
- Java는 JAR 로드 후 파일 핸들을 해제하므로, 실행 중 덮어쓰기 가능 (Linux/Windows 동일)
- 만약 Windows에서 파일 잠금 발생 시, `new_mwagent.jar` 방식으로 전환 가능 (`restart.bat`에 해당 로직 내장됨)
