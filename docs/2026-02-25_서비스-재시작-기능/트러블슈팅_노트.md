# MWAgent 재시작 기능 구현 트러블슈팅 노트

> 작성일: 2026-02-25  
> 대상 파일: `run.agent.bat`, `restart.bat`

---

## 문제 1: `. was unexpected at this time.` (1차)

### 증상
```
C:\Users\SYSADMIN\MWAgent>run.agent.bat
. was unexpected at this time.
```

### 원인
`schtasks /TR` 인자로 넘기는 문자열 안에 `%~dp0`를 사용했는데,  
`%~dp0`가 `C:\Users\SYSADMIN\MWAgent\`로 확장될 때 **끝의 `\`가 바로 뒤의 `"`를 이스케이프** 처리함.

```bat
:: 문제 코드
set RESTART_CMD=cmd /c "cd /d %~dp0 && restart.bat"
schtasks /Create /TR "%RESTART_CMD%" ...
```

`%~dp0` = `C:\Users\SYSADMIN\MWAgent\` 이므로  
`RESTART_CMD` 값이 `cmd /c "cd /d C:\Users\SYSADMIN\MWAgent\ && restart.bat"`  
→ 끝의 `\"`가 이스케이프되어 `"` 닫힘이 깨짐 → 파서 에러 발생

### 해결
`schtasks /TR`에 복잡한 cmd 명령 대신 **배치 파일 절대 경로를 직접 지정**.

```bat
:: 수정 코드
schtasks /Create /TN "MWAgentRestart" /TR "%~dp0restart.bat" ...
```

`%~dp0restart.bat` = `C:\Users\SYSADMIN\MWAgent\restart.bat` 로 깔끔하게 확장됨.

## 문제 2: `. was unexpected at this time.` (2차 — BOM 문제)

### 증상
문제 1 수정 후에도 동일한 에러 재발.  
파일 내용(type 명령)은 정상인데 실행하면 에러.

### 원인
**UTF-8 BOM (Byte Order Mark)** 문제.  
파일 편집 도구가 파일 앞에 `EF BB BF` (BOM) 3바이트를 삽입.  
CMD 파서가 이 바이트를 `.` 문자로 인식하여 `@echo off` 이전에 `.` 명령을 실행하려 시도.

```
파일 내부 (헥스):
EF BB BF 40 65 63 68 ...
↑BOM    ↑@echo...
```

### 확인 방법
```powershell
$bytes = [System.IO.File]::ReadAllBytes('run.agent.bat')
$bytes[0..2] | ForEach-Object { "{0:X2}" -f $_ }
# BOM 없으면: 40 65 63 (= @ec)
# BOM 있으면: EF BB BF
```

### 해결
PowerShell `StreamWriter`로 **BOM 없는 ASCII** 인코딩으로 파일 저장.

```powershell
$sw = New-Object System.IO.StreamWriter('경로\run.agent.bat', $false, [System.Text.Encoding]::ASCII)
# 내용 작성
$sw.Close()
```

> ⚠️ 주의: 이 방식으로 저장한 파일에는 한글 주석이 깨짐. **주석은 영문으로 작성**해야 함.

## 문제 3: `Service restart failed` 로그

### 증상
```
[RESTART] Service stopped.
[RESTART] No JAR update. Restart only.
[RESTART] Service restart failed.   ← net start가 실패
```

로그에는 "Service stopped"로 찍히나, 작업 관리자에서 `java.exe`가 여전히 실행 중.

### 원인
`net stop MWAgent`가 **WinSW 서비스 래퍼만 중지**하고,  
실제 Java 프로세스(`java.exe`)는 아직 살아있음.  
그 상태에서 `net start MWAgent`를 하면 리소스 충돌로 실패.

```
Windows 서비스 (MWAgent)
    └── mwagent-service.exe  ← 서비스 래퍼 (WinSW)
            └── java.exe     ← 실제 MWAgent (이게 안 죽음)
```

### 해결
서비스 중지 전 `sc queryex MWAgent`로 **java.exe의 PID를 확보** 후,  
종료 후 `taskkill /F /PID`로 직접 강제 종료.

```bat
:: PID 확보
for /f "tokens=3" %%a in ('sc queryex MWAgent ^| find "PID"') do set SERVICE_PID=%%a

:: 비동기 중지 신호 전송
start /b sc stop MWAgent

:: 대기 후 PID로 강제 종료
ping 127.0.0.1 -n 4 > nul
taskkill /F /PID %SERVICE_PID% > nul 2>&1
```

## 문제 4: `sc stop` 블로킹으로 스크립트 멈춤

### 증상
```
[INFO] Stopping MWAgent Service...
         ← 여기서 스크립트가 멈춤 (무한 대기)
```

### 원인
`sc stop MWAgent` 명령은 서비스가 완전히 STOPPED 상태가 될 때까지 **블로킹(대기)** 함.  
서비스가 STOP_PENDING 상태에서 멈춰 있으면 스크립트 전체가 영원히 대기.

> 참고: `net stop`도 동일하게 블로킹됨.  
> `> nul 2>&1`은 출력만 숨길 뿐, 블로킹을 막지 않음.

### 해결
`start /b`로 **백그라운드 비동기 실행** 후, 바로 `taskkill`로 프로세스 직접 종료.

```bat
:: 비동기로 중지 신호만 보내고 기다리지 않음
start /b sc stop MWAgent
ping 127.0.0.1 -n 4 > nul

:: PID로 직접 강제 종료 (블로킹 없음)
taskkill /F /PID %SERVICE_PID% > nul 2>&1
```

## 문제 5: `[SC] OpenService FAILED 5: Access is denied.`

### 증상
```
[INFO] Sending stop signal...
[SC] OpenService FAILED 5: Access is denied.
```

### 원인
작업 스케줄러가 `restart.bat`을 **일반 사용자(SYSADMIN) 권한**으로 실행.  
`sc stop/start MWAgent`는 **관리자 권한**이 필요한 명령.

### 해결
`run.agent.bat`에서 작업 스케줄러 등록 시 **`/RU SYSTEM`** 옵션 추가.  
SYSTEM 계정은 서비스 제어 권한을 포함한 최고 권한 보유.

```bat
schtasks /Create /TN "MWAgentRestart" /TR "%~dp0restart.bat" ^
    /SC ONCE /ST 00:00 /SD 01/01/2000 /RU SYSTEM /F
```

> ⚠️ `/RU SYSTEM` 작업 등록 자체는 **관리자 권한 필요**.  
> 운영 환경에서는 MWAgent가 SYSTEM 계정으로 서비스 실행되므로 자동으로 충족됨.  
> 수동 테스트 시에는 반드시 **관리자 CMD**에서 실행.

## 문제 6: `. was unexpected at this time.` (3차 — if 블록 + AutoRun)

### 증상
파일 내용, BOM 모두 정상인데 `cmd /c`로 실행하면 에러,  
`cmd /d /c`(AutoRun 비활성화)로 실행하면 정상.

### 원인
`if` 블록 안에서 `%date% %time%`를 직접 사용할 경우,  
시스템의 **CMD AutoRun** 레지스트리 설정이 날짜/시간 형식을 변조하거나  
파싱 과정에서 특수문자 충돌 발생.

```bat
:: 문제 코드 (if 블록 안에서 직접 사용)
if "%~1"=="" (
    echo [INFO] Restart only.
    echo [%date% %time%] ... >> %LOG_FILE%   ← CMD AutoRun 환경에서 파싱 충돌
)
```

### 해결
`if` 블록 진입 **전에** `%date% %time%`를 변수로 캡처.

```bat
:: 수정 코드
set NOW=%date% %time%   ← 블록 밖에서 미리 캡처

if "%~1"=="" (
    echo [INFO] Restart only.
    echo [%NOW%] ...     ← 이미 확정된 문자열 사용
)
```

## 최종 동작 흐름 (정상)

```
[관리자 CMD 또는 SYSTEM 서비스]
  run.agent.bat 실행
    → sc queryex로 java PID 확보
    → schtasks /Create /RU SYSTEM으로 restart.bat 등록
    → schtasks /Run으로 즉시 실행

  [SYSTEM 권한으로] restart.bat 실행
    → start /b sc stop MWAgent  (비동기, 블로킹 없음)
    → taskkill /F /PID {java PID}  (강제 종료)
    → ping으로 3초 대기
    → net start MWAgent  (서비스 재시작)
    → schtasks /Delete  (작업 정리)
```
